# about-me.md

## Role
Founder / product builder — non-technical but product-minded.

## Coding Comfort
Can read code but cannot write it. Can follow along, spot obvious issues, but rely on AI to author and debug.

## Computer
Mac

## How I Like to Work

**Responses:** Short and direct. Skip the preamble. Tell me what you did or what you need.

**Big changes:** Just do it. Don't ask for permission on implementation details — make the call.

**When something is unclear:** Make your best guess, tell me what you assumed, move on.

**After making changes:** Just list the files changed. No need to explain every line.

**Testing:** Skip tests for now — move fast in Phase 1, add tests before launch.

## Things That Slow Me Down
- Long explanations before doing the thing
- Asking clarifying questions that could just be a reasonable assumption
- Showing me multiple options when one is clearly better

## Keep This File
This file is about me, not the project. Copy it to every new project.
